### 1. ReLU는 선형 직선들의 조합인데 어떻게 '비선형성'을 학습하는가? (수학적 구조)
단순 선형 조각의 연결일 뿐인 ReLU가 어떻게 복잡한 고차원의 비선형 곡면을 학습하는지 규명하기 위해 본 실험을 설계했습니다. 실험 결과, 레이어가 깊어짐에 따라 활성화되는 뉴런의 조합이 영역별로 다차원적으로 구겨지고 뒤틀리는 '꺾임(Folding)' 현상을 목격했습니다. 이는 마치 미적분에서 곡선의 면적을 직사각형들로 쪼개어 근사하는 것처럼, 무수한 선형 Knot들이 이어 붙여지며 어떤 복잡한 비선형 경계면도 완벽히 흉내 낼 수 있는 '조각별 선형 함수(Piecewise Linear Function)'를 완성해 내는 핵심 메커니즘임을 실증했습니다.

### 2. 르쿤 초기화(LeCun Initialization)의 역설: 어떻게 출력측(fan-out)을 배제하고도 분산을 보존하는가? (순방향 전파)
양방향의 스케일을 모두 맞추는 글로럿과 달리, 오직 입력 뉴런 수($\text{fan}_{\text{in}}$)만 고려하는 르쿤 초기화가 어떻게 분산을 사수하는지 수학적 맹점을 추적했습니다. 수식 유도와 실험을 통해 순방향 전파 시 현재 레이어의 출력 분산에 영향을 주는 항의 개수는 오직 직전 입력의 개수뿐이며, 다음 층의 뉴런 수는 현재 스케일에 간섭하지 못한다는 독립성을 확인했습니다. 즉, 역방향 그라디언트 분산까지 타협한 글로럿과 달리 순방향 분산 보존이라는 단일 목적에 화력을 집중함으로써 심층망에서도 신호가 파괴되지 않음을 실증했습니다.

### 3. AdaGrad의 학습률 조정 메커니즘: 왜 가파른 축의 학습률을 줄이는 게 최적점으로 인도하는가? (지형 평탄화)
가파른 절벽을 만나면 더 빠르게 내려가야 한다는 직관을 깨고, AdaGrad가 왜 가파른 축의 발걸음에 브레이크를 거는지 기하학적 궤적을 분석했습니다. 경사가 가파른 축은 조금만 보폭이 커도 최적점을 지나쳐 반대편 벽으로 튕겨 나가는 지그재그 진동(Oscillation)을 유발하며 에너지를 낭비하게 됩니다. AdaGrad는 누적 그라디언트로 이 가파른 성분의 스케일을 깎아내어, 뒤틀린 대각선 지형에서도 그라디언트 벡터의 방향을 진동축이 아닌 실제 최적점이 있는 중심 방향으로 꺾어주는 공간 정형화(Preconditioning)를 수행함을 실증했습니다.

### 4. ReduceLROnPlateau의 철학: 에러가 줄어들지 않을 때, 왜 항상 학습률을 줄이기만 할까? (수렴 최적화)
손실이 줄어들지 않는 정체기(Plateau)를 만났을 때, 웅덩이를 탈출하기 위해 학습률을 일시적으로 키워야 하지 않느냐는 의문에서 출발해 고차원 공간의 실체를 파고들었습니다. 고차원 오차 지형에서 정체기는 모델이 멈춘 것이 아니라, 발걸음이 너무 커서 최적점 문앞을 격렬하게 맴도는 오버슈팅(Overshooting) 상태임을 확인했습니다. 현대 딥러닝에서 완벽히 막힌 로컬 미니멈에 갇힐 확률은 제로에 수렴하므로, 정체기에서의 학습률 삭감은 진동을 멈추고 최적점의 가장 깊은 바닥으로 안착시키는 세밀한 마침표 전략임을 실증했습니다.

### 실험 1. ReLU 투영을 통한 비선형 곡선 함수(Sine) 조각별 선형 근사 검증
* **파일명:** `01_relu_piecewise_linear_approximation.py`
* **실험 목적 (Why):** 각진 선형 결합인 ReLU 뉴런들이 레이어를 통해 쌓였을 때, 실제 연속적인 비선형 타겟 곡선을 어떻게 영역별로 쪼개어 정교하게 근사해 내는지 눈으로 확인합니다.
* **실험 방법 (How):** 비선형 타겟 데이터($y = \sin(x)$)를 생성한 후, ReLU 활성화 함수를 가진 얕은 MLP 모델을 구축하여 의도적으로 꺾이는 지점(Knot)들이 유기적으로 연결되며 곡선을 흉내 내는 과정을 그래프로 시각화합니다.
* **실험 결과 (Result):** 완벽한 곡선 수식이 아님에도 불구하고, 미세하게 각진 직선 조각들이 촘촘하게 이어지며 사인파 형상을 완벽하게 추종하는 조각별 선형 함수의 기하학적 메커니즘을 증명했습니다.

### 실험 2. 가중치 초기화에 따른 깊은 신경망의 출력층 분산 보존 정량 평가
* **파일명:** `02_lecun_initialization_variance_tracking.py`
* **실험 목적 (Why):** 일반 무작위 정규분포 초기화와 르쿤 초기화가 10층 이상의 딥 네트워크에서 순방향 전파 시 뉴런 출력의 분산을 제어하는 능력을 비교 벤치마킹합니다.
* **실험 방법 (How):** 입력 데이터 분산을 1.0으로 고정한 후 500차원의 선형 레이어를 10층까지 깊게 쌓아 통과시킵니다. 일반 무작위 초기화(std=1.0)와 르쿤 초기화($\text{std}=\sqrt{1/\text{fan}_{\text{in}}}$)를 각각 적용하여 층별 출력값의 분산 변화를 로그 스케일로 추적합니다.
* **실험 결과 (Result):** 일반 초기화는 층이 깊어질수록 분산이 무한대로 폭발하여 연산이 파괴된 반면, 르쿤 초기화는 10층이 지나도 출력 분산이 대략 1.0 안팎의 일정한 스케일을 유지하며 순방향 전파의 완벽한 스케일 사수 능력을 증명했습니다.

### 실험 3. 비대칭 경사 지형에서 옵티마이저(SGD vs AdaGrad) 최적화 경로 추적
* **파일명:** `03_adagrad_asymmetric_valley_trajectory.py`
* **실험 목적 (Why):** $y$축의 경사가 $x$축보다 극단적으로 가파른 타원형 손실 지형에서 AdaGrad가 어떻게 진동을 억제하고 안정적으로 최적점에 안착하는지 경로(Trajectory)를 시각화합니다.
* **실험 방법 (How):** 가상의 비대칭 골짜기 손실 함수를 정의하고 동일한 시작점에서 출발시킵니다. 고정 학습률을 가진 일반 SGD와 적응형 학습률을 가진 AdaGrad의 매 스텝 매개변수 이동 궤적을 2차원 등고선(Contour Map) 위에 플롯하여 대조합니다.
* **실험 결과 (Result):** SGD는 가파른 벽 사이를 미친 듯이 지그재그로 오버슈팅하며 튕겨 나갔으나, AdaGrad는 가파른 축의 학습률 분모가 커지면서 진동 에너지를 급격히 감쇄시키고 최적점 $(0,0)$으로 곧바로 꺾여 들어가는 안정적인 공간 정형화 효과를 실측했습니다.

### 실험 4. 손실 정체기(Plateau)에서 학습률 삭감에 따른 수렴도 하강 실증
* **파일명:** `04_learning_rate_plateau_loss_decay.py`
* **실험 목적 (Why):** 모델 학습 중 손실(Loss) 하강이 멈춘 정체기에서 학습률을 감소시키는 스케줄러 전략이 실제로 유효한 돌파구가 되는지 시나리오를 통해 실증합니다.
* **실험 방법 (How):** 노이즈가 강한 합성 데이터셋과 분류 신경망을 구축한 뒤 큰 학습률(0.5)로 학습을 시작합니다. `ReduceLROnPlateau` 스케줄러를 연동하고, Epoch에 따른 손실값의 변화 곡선과 실제 적용된 학습률의 변화 곡선을 TwinX 축 그래프로 매핑하여 관찰합니다.
* **실험 결과 (Result):** 손실 곡선이 수평을 그리며 완벽한 정체기(Plateau)에 빠져있다가, 스케줄러가 이를 감지하여 학습률을 0.1배로 뚝 떨어뜨린 동기화 지점에서 손실값이 다시 한 단계 아래 바닥(Global Minimum)으로 급격히 미끄러져 내려가는 마침표 효과를 실측했습니다.

핵심 소스코드 아키텍처

### 1. 비대칭 골짜기 지형 최적화 경로 비교 모듈 (`03_adagrad_asymmetric_valley_trajectory.py`)

```python
# 1. y축 경사가 x축보다 20배 가파른 비대칭 손실 골짜기 수식 설계
def loss_func(x, y):
    return 1.0 * (x ** 2) + 20.0 * (y ** 2)

# 2. 동일한 가파른 벽 위 시작점에서 각각의 옵티마이저 구동
def simulate_optimizer(opt_name, lr=0.1, steps=30):
    w = torch.tensor([4.0, 4.0], requires_grad=True)
    
    if opt_name == 'sgd':
        optimizer = torch.optim.SGD([w], lr=lr)
    elif opt_name == 'adagrad':
        # AdaGrad는 자체적으로 분모에서 경사를 깎으므로 상대적으로 큰 lr 수용 가능
        optimizer = torch.optim.Adagrad([w], lr=lr)
        
    path = []
    for _ in range(steps):
        path.append(w.detach().clone().numpy())
        loss = loss_func(w[0], w[1])
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        
    return np.array(path)
